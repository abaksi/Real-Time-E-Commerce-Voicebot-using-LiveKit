Selected Project: Real-Time E-Commerce Voicebot using LiveKit
Voice-based interfaces are becoming an important channel for customer interaction in digital commerce, enabling faster support, hands-free access, and improved accessibility. Building such systems requires not only conversational intelligence but also reliable real-time audio streaming and low-latency response handling. In this capstone, you will design and implement a real-time AI voicebot for an e-commerce platform that can interact with customers through spoken conversation, answer common queries, provide basic product information, and assist with order tracking.
Objectives
•	Build a real-time voice interaction system using LiveKit and WebRTC.
•	Integrate speech-to-text (STT) and text-to-speech (TTS) components into a live conversational loop.
•	Use an LLM-driven conversational engine to understand and respond to user queries.
•	Implement basic decision logic to route user queries to appropriate responses or backend actions.
•	Demonstrate a functional end-to-end voicebot suitable for an e-commerce use case.
________________________________________
Core Concepts & Techniques
•	Large Language Models (LLMs) for conversational reasoning.
•	System prompting to define assistant role, tone, and behavior.
•	Real-time AI pipelines (audio → text → reasoning → speech).
•	Optional Retrieval-Augmented Generation (RAG) for factual consistency.
•	Tool / function calling for backend actions (e.g., order status lookup).
•	Handling uncertainty, errors, and unsupported requests in voice interfaces.
________________________________________
Suggested Tools / Frameworks
•	Real-Time Audio Streaming: LiveKit (WebRTC)
•	Backend: Python or Node.js
•	LLM: Any API-based or local LLM
•	Speech-to-Text (STT): Whisper or LiveKit transcription
•	Text-to-Speech (TTS): Any TTS engine (cloud or local)
•	Frontend: Web-based voice interface using LiveKit SDK
